## Description

REST API for **M&M Trolley** (locations, time slots, bookings, booking policy). Matches the frontend contract in `m-m-trolley-web` (JWT after password codes, same business rules as the mock).

## Environment

Copy `.env.example` to `.env` and set `DATABASE_URL`, `JWT_SECRET`, optional `FRONTEND_ORIGIN` (comma-separated origins allowed).

## Database

Uses **Prisma 7**: `DATABASE_URL` is read from **`prisma.config.ts`** (CLI / Migrate). At runtime, `PrismaService` uses **`@prisma/adapter-pg`** (no `url` in `schema.prisma` — that is required by Prisma 7 + the VS Code extension).

```bash
npx prisma migrate deploy
# or during development:
npx prisma migrate dev
```

## API base URL

All routes are under **`/api/v1`**.

| Method | Path | Auth | Notes |
|--------|------|------|--------|
| POST | `/auth/login` | Public | `{ "role": "admin" \| "participant", "accessCode" }` → `{ role, accessToken }` |
| GET | `/booking-policy` | Public | `{ isGlobalLocked, lockedLocationIds }` |
| PATCH | `/booking-policy/global` | Admin + Bearer | `{ "isLocked": boolean }` |
| PATCH | `/booking-policy/locations/:locationId` | Admin + Bearer | `{ "isLocked": boolean }` |
| GET | `/locations` | Bearer | Admin + participant |
| GET | `/locations/:locationId/schedule` | Bearer | |
| POST | `/locations` | Admin | Create place |
| PATCH | `/locations/:locationId` | Admin | Update (published places: status-only) |
| DELETE | `/locations/:locationId` | Admin | Draft only |
| POST | `/locations/:locationId/slots` | Admin | `{ startTimeIso, endTimeIso, capacity }` |
| POST | `/slots/:timeSlotId/clear` | Admin | |
| DELETE | `/slots/:timeSlotId` | Admin | |
| POST | `/bookings/reserve` | Participant + Bearer | `{ timeSlotId, participantName, seats }` |
| POST | `/bookings/admin` | Admin + Bearer | Manual add |
| DELETE | `/bookings/:bookingId` | Admin | |

## Project setup

```bash
$ npm install
$ npx prisma generate
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
